Angular Animation

Angular contains an Animation DSL language that allows for complex animations to defined for components.

To get started, just include BrowserAnimationsModule into your application module.

The DSL works atop of the web-animations API and requires a polyfill.

Triggers

Aniamtions in Angular are linked in the template using @properties.

When a property changes state (or the element is inserted/destroyed), it will trigger an animation.

The @trigger is bound to the component (that's where the animation is defined).

Component Animations

The animation itself is defined on the component using the DSL language.

Triggers kick off transitions and a transition consists of animation steps.

When an animation is complete, it will jump to a state and the final styles will be applied.

animate() and style()

The animate() and style() steps are the bread and butter of animations in Angular.

style() applies the styling to the element immediately.

animate('time', style(...)) applies the styling to the over a specified duration.

animate('time', keyframes(...)) applies a series of keyframes over a specified duration.

Insertion / Removal

When an item is inserted and removed from a view it has special :enter and :leave animations.

:enter is run when any *ngIf, *ngFor, *ngSwitch item is placed into the page.

:leave is run when they are removed from the page.

(@animation.callback) code

When an animation starts and finishes it will emit callbacks

(@trigger.start) is called on start and (@trigger.done) when the animation completes

An AnimationEvent is passed back via $event in the callback
